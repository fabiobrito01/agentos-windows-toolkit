# Política de segurança

## Relatar uma vulnerabilidade

Não publique credenciais, dados pessoais ou detalhes exploráveis em uma Issue. Use **Security → Report a vulnerability** no GitHub para comunicar o problema de forma privada.

Inclua a ferramenta afetada, versão do Windows, impacto observado e passos mínimos para reprodução. Não inclua senhas, tokens, chaves privadas, endereços internos nem arquivos pessoais.

## Modelo de confiança

Os scripts são legíveis e executados localmente. Revise o conteúdo antes de usar, baixe somente deste repositório e não execute cópias recebidas de fontes desconhecidas. Ferramentas administrativas são identificadas no catálogo, mas essa indicação não substitui uma revisão do comando no seu ambiente.
