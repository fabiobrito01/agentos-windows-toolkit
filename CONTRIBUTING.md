# Como contribuir

Contribuições são bem-vindas por Issues e Pull Requests.

1. Crie um fork e uma branch curta.
2. Não inclua caminhos absolutos, nomes de usuário, IPs privados reais, credenciais ou dados pessoais.
3. Aceite caminhos por argumento ou pergunte ao usuário; use `%USERPROFILE%` apenas como padrão seguro.
4. Valide entradas e peça confirmação explícita antes de excluir, sobrescrever ou alterar o sistema.
5. Documente objetivo, requisitos, nível de risco, efeitos e forma de reversão.
6. Teste em uma máquina virtual quando o script alterar Registro, serviços, rede ou arquivos do sistema.

Ao abrir o PR, descreva o que mudou, por que mudou, como foi testado e qual o impacto para o usuário.
